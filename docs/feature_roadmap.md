## F-01: 月間計画（月→週連携）

### 目的
月単位でプロジェクトタスクの作業計画を立て、週間予定に自動展開する。
**月→週→日** の3階層計画を実現し、計画精度の向上と手入力の削減を図る。

### 概要イメージ
```
┌─────────────────────────────────────────────────────────┐
│  📅 月間計画 — 2026年4月                                 │
├─────────────────────────────────────────────────────────┤
│  タスク名         │ W1(4/1〜) │ W2(4/6〜) │ W3(4/13〜) │ W4(4/20〜) │
│ ─────────────────┼──────────┼──────────┼───────────┼───────────│
│ ・AI開発           │          │          │           │           │
│   ChatBot実装     │   8h     │  12h     │   8h      │   4h      │
│   プロンプト設計   │   4h     │   4h     │   0h      │   0h      │
│ ・Web開発          │          │          │           │           │
│   フロント改修     │   0h     │   4h     │  12h      │  12h      │
│ ─────────────────┼──────────┼──────────┼───────────┼───────────│
│ 合計              │  12h     │  20h     │  20h      │  16h      │
└─────────────────────────────────────────────────────────┘
          ↓ 「週予定に展開」ボタン
    weekly_schedule に自動登録
```

### 設計案

#### DBテーブル: `monthly_plan`
| カラム | 型 | 概要 |
|--------|-----|------|
| id | INTEGER PK | 自動採番 |
| user_id | INTEGER FK | ユーザーID |
| year_month | TEXT | 対象年月（YYYY-MM） |
| week_number | INTEGER | 月内の週番号（1〜5） |
| week_start | TEXT | 週開始日（YYYY-MM-DD） |
| project_task_id | INTEGER FK | プロジェクトタスクID（任意） |
| task_name | TEXT | 作業名（手入力も可） |
| planned_hours | REAL | 計画時間 |
| category_id | INTEGER FK | 大区分ID |
| subcategory_id | INTEGER FK | 中区分ID |
| created_at | TEXT | 作成日時 |
| updated_at | TEXT | 更新日時 |

#### 画面: `/monthly-plan`
- 月選択（前月・次月ナビ）
- タスク × 週のマトリックス入力（時間）
- タスク管理（project_task）から候補を選択 or 手入力
- 「週予定に展開」ボタン → 選択した週のweekly_scheduleにAM/PM自動配分
- 月合計・週合計の自動集計

#### 週予定への展開ロジック
```
1. 月間計画から対象週のタスク・時間を取得
2. 既存のweekly_scheduleと重複チェック
3. AM/PM × 5枠に自動配分（空き枠に順番に入れる）
4. 配分ルール：
   - AM優先 → PM へ流す
   - 1日あたり上限 = std_hours（ユーザーの基本時間）
   - 入りきらない場合は警告を出す
```

#### 権限
| ロール | 操作 |
|--------|------|
| ユーザー | 自分の月間計画を作成・編集 |
| 管理職 | 自部署メンバーの月間計画を閲覧・編集 |
| マスタ | 全メンバーの月間計画を閲覧・編集 |

## F-03: タスク管理 ↔ 週間予定の連携

### 目的
タスク管理（project_task）と週間予定（weekly_schedule）を紐付け、日次実績の時間をタスクの進捗に自動反映する。

### 概要
```
project_task（タスク管理）
    ↕ 紐付け
weekly_schedule / daily_result（週間予定 / 日次実績）
    ↓ 自動集計
project_task.actual_hours（実績時間）
    → 進捗率に反映
```

### 設計案
- `weekly_schedule` に `project_task_id` カラムを追加
- 日次実績保存時に紐付いたproject_taskの実績時間を自動集計
- タスク管理画面に「計画時間」「実績時間」列を追加

### 実装タスク
- [ ] DB — `weekly_schedule` に `project_task_id` 追加
- [ ] 週間予定画面 — タスク選択時にproject_task候補を表示
- [ ] 実績集計 — daily_resultからproject_taskへの時間集計

プロンプト設計    ■■■■■■░░░░
